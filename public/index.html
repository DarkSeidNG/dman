    Welcome to the dman API

    Welcome to the dman API - A Document Management System API! You can use our API to access dman API endpoints, which gives you information on how to register, login, add a new document and manage your document in our database.

    We have language bindings in JavaScript! You can view code examples in the dark area to the right. This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API documentation(s).

    Authentication

    An authorization token is required to access all routes except the Register and Login routes. Try the Register or Login route to get a token.

    A token generated expires 72 hours after it is being generated.

    The header should be set in the following format:

    Authorization: token

    You must replace thetoken with the one gotten after authentication.

    Errors

    The dman API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your email or password is wrong.
    403 Forbidden -- The document/user requested is hidden for administrators only.
    404 Not Found -- The specified document/user could not be found.
    405 Method Not Allowed -- You tried to access a document/user with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    409 Conflict -- Duplicate record.
    410 Gone -- The document/user requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many documents/users! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
